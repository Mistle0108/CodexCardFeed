# CodexCardFeed

Codex 대화를 버려지는 출력이 아니라 재사용 가능한 지식 자산으로 정리하는 로컬 도구다.

## 왜 만들었는가

Codex 대화는 많은 토큰 비용을 들여 만들어지지만, 대부분은 그 순간의 작업이 끝나면 가치가 급격히 떨어지는 일회성 출력으로 남는다. 이미 생산된 대화 결과물 안에는 문제 해결 과정, 판단 근거, 반복해서 참고할 수 있는 질문과 답변이 들어 있지만, 시간이 지나면 다시 찾기 어렵고 결국 같은 질문을 반복하게 된다.

이 프로젝트는 그렇게 소비되고 사라지는 Codex 대화를 프로젝트, 스레드, 턴 단위로 수집하고 정리해, 버려지는 결과물이 아니라 다시 찾고 분류하고 관리할 수 있는 자료이자 재사용 가능한 지식 자산으로 전환하는 것을 목표로 한다.

## 무엇을 해결하는가

CodexCardFeed는 Codex 안에 흩어져 있는 대화를 별도의 로컬 UI에서 다시 구조화한다. 사용자는 프로젝트, 채팅, 스레드, 턴 단위로 대화를 탐색하고, 카드 형태로 핵심 내용을 빠르게 훑고, 필요할 때 원본 Codex 스레드로 돌아갈 수 있다.

핵심은 새로운 내용을 생성하는 것이 아니라, 이미 토큰을 써서 만들어진 결과물을 다시 활용 가능한 형태로 정리하는 데 있다.

## 현재 할 수 있는 것

- Codex 로컬 세션 import
- 프로젝트와 채팅 구분 표시
- 스레드 목록 탐색
- 턴 카드 목록 보기
- 질문만 따로 모아보기
- 턴 상세 보기
- Codex 원본 스레드 열기
- Codex source path와 Database path 확인 및 변경

## 어떻게 동작하는가

CodexCardFeed는 Codex의 로컬 대화 데이터를 읽어 필요한 메타데이터를 추출하고, 이를 로컬 SQLite 데이터베이스에 저장해 빠르게 탐색할 수 있도록 한다. 사용자는 Electron 기반 UI에서 이 데이터를 프로젝트, 스레드, 턴 구조로 탐색한다.

원본 Codex 데이터는 직접 수정하지 않고, 이 프로그램 안에서만 별도의 관리 레이어를 둔다. 즉 원본은 유지하고, 정리와 탐색을 위한 구조만 따로 만든다.

## 핵심 개념

### Project

Codex에서 하나의 작업 맥락으로 묶이는 상위 단위다. 여러 스레드를 포함할 수 있다.

### Thread

하나의 대화 흐름이다. 사용자가 사이드바에서 선택하는 일반적인 채팅 단위에 가깝다.

### Turn

한 번의 질문과 그에 이어지는 작업 및 최종 응답을 묶는 단위다. 이 앱에서는 카드의 기본 단위로 사용된다.

### Item

턴 내부에 저장되는 세부 기록 단위다. 사용자 메시지, 어시스턴트 진행 메시지, 최종 답변, 도구 호출 같은 실제 저장 이벤트가 여기에 해당한다.
