# CodexCardFeed 기획

## 한 줄 정의

`CodexCardFeed`는 Codex에 쌓인 대화를 카드형으로 정리하고, 태깅과 검색을 통해 과거 세션을 다시 찾아 쓸 수 있게 만드는 로컬 탐색 도구다.

## 문제 정의

Codex를 계속 사용하다 보면 예전에 이미 했던 질문이나 작업을 다시 반복하게 된다.

이유는 다음과 같다.

- 대화가 계속 누적된다.
- 과거 세션을 빠르게 훑을 수 없다.
- 비슷한 주제의 대화를 다시 찾기 어렵다.
- 이전에 해결한 내용을 재사용하기보다 같은 질문을 다시 하게 된다.

## 기획 목적

Codex의 대화 내용을 `프로젝트 / 세션 / 턴` 단위로 구조화해서 보여주고, 카드형 UI로 한눈에 탐색할 수 있게 만든다.

핵심은 새로운 답을 생성하는 것이 아니라, 이미 존재하는 대화를 다시 찾고 재사용하기 쉽게 만드는 것이다.

## 해결 방향

- 대화 내용을 카드형으로 묶어 시각적으로 정리한다.
- 각 카드에는 질문과 답변의 일부만 짧게 보여준다.
- 필요할 때 턴 상세 내용을 열어 원문을 확인한다.
- 태깅과 인덱싱을 통해 세션 간 검색이 가능하도록 만든다.
- 특정 태그나 문구에 대한 모아보기를 제공한다.

## 요약 기능 제외 이유

초기에는 사용자 API를 연결해 자동 요약을 붙이는 방안을 검토했다.

하지만 OpenAI API는 별도 과금 구조이고, 현재 사용 중인 요금제에서는 현실성이 낮다.

따라서 현재 방향에서는 API 기반 요약을 제외하고, 대신 아래 기능에 집중한다.

- 태그
- 인덱싱
- 검색
- 모아보기

## 핵심 기능

- Codex 로컬 세션 데이터를 읽어와 구조화
- `프로젝트 / 세션 / 턴` 단위 탐색
- 질문/답변 일부를 보여주는 카드형 목록
- 카드에서 상세 보기 팝업 제공
- 태그 추가
- 검색어 기반 세션/턴 검색
- 특정 태그나 문구에 대한 모아보기
- 원본 대화는 수정하지 않고 별도 앱에서만 관리

## 검색 기능 역할

검색은 단순 텍스트 찾기만을 의미하지 않는다.

목표는 다음과 같다.

- 예전에 어떤 맥락에서 어떤 작업을 했는지 다시 찾기
- 비슷한 주제의 대화를 묶어서 보기
- 이미 해결한 내용을 다시 재사용하기

즉, 검색 결과는 문구 하나만 찾는 것이 아니라 관련 턴과 세션까지 함께 탐색할 수 있는 형태를 지향한다.

## UX 목표

- 과거 대화를 빠르게 훑을 수 있어야 한다.
- 같은 질문을 반복하는 일을 줄여야 한다.
- 찾은 대화를 바로 다시 열어 맥락을 이어갈 수 있어야 한다.
- 정리 도구 자체가 무겁거나 번거롭지 않아야 한다.

## MVP 범위

- 세션 import
- 카드형 목록 UI
- 턴 상세 보기
- 태깅
- 검색
- 태그/문구 모아보기

## 비목표

- AI 요약 생성
- 원본 Codex 대화 수정
- 실시간 동기화
- 복잡한 협업 기능

## 기대 효과

이 도구를 통해 사용자는 Codex를 단순한 대화창이 아니라, 재사용 가능한 작업 기록 저장소처럼 다룰 수 있게 된다.

결과적으로 다음 효과를 기대한다.

- 반복 질문 감소
- 이전 작업 맥락 재활용
- 세션 간 탐색 효율 향상
- 개인 작업 기록 정리 비용 감소
